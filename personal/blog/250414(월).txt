# 1. 프로세스 스케줄링
운영체제(OS) 입장에서는 CPU는 하나인데 실행해야할 프로세스(작업)은 많다. 그래서 CPU의 자원을 효율적이고 공정하게 분배해서 시스템을 원활하게 동작시키기 위해 프로세스 스케줄링을 진행한다.

## 2. 목적
프로세스 스케줄링을 진행하는 목적은 아래와 같다. 

1. CPU 효율 극대화
→ CPU가 최대한 쉬지 않고 계속 일을 하게 만들어 시스템 성능을 높이기 위해서

2. 공정성 (Fairness)
→ 특정 프로세스만 계속 CPU를 독점하는 걸 막고, 모든 프로세스가 적절하게 CPU를 사용할 기회를 갖도록

3. 응답 시간 최소화
→ 사용자의 요청에 최대한 빠르게 반응하도록 (특히 인터랙티브 시스템)

4. 처리량(Throughput) 증가
→ 단위 시간당 처리할 수 있는 프로세스의 수를 늘리기 위해

5. 대기 시간 최소화
→ 프로세스가 CPU 할당을 기다리는 시간을 최소화

6. 우선순위 고려
→ 긴급하거나 중요한 프로세스에 더 빠르게 CPU를 배정하기 위해

## 3. 종류
프로세스 스케줄링은 크게 선점형과 비 선점형으로 나뉜다. 여기서 "선점형"이란 프로세스가 CPU를 사용하고 있더라도, 운영체제가 **강제로** 프로세스의 CPU 사용을 중단시키고 다른 프로세스에게 CPU를 넘길 수 있다는 것을 의미한다. _~~종류에 대한 설명이 다소 개인적일 수 있다.. ㅎ~~_

1) 비선점형 스케줄링 
- SJF (Shortest Job First) : 실행시간이 짧은 프로세스 먼저 할당
- FCFS (First Come First Service) : 먼저 들어온 것 먼저 처리
- HRN : "경로 우대 사상" (대기시간 + 실행시간) / 실행시간 = 높은 값이 우선순위를 가진다.
- 기한부 : 기한을 정해놓고 다 수행하지 못했을 경우 다시 실행
- 우선순위

2)  선점형 스케줄링
- SRT (Shortest Remaining Time) : 남은 작업 시간이 짧은 프로세스로 교체
- RR(Round Robin) : 타임 슬라이스를 할당해서 시간대별로 스케줄링
- MLQ(다단계 큐) : 프로세스가 들어왔을 때, 그게 맞은 큐를 할당하는 스케줄링
- MLFQ(다단계 피드백 큐) : 피드백이 있는 다단계 큐. 실행상황에 따라 큐를 올렸다 내렸다 하면서 프로세스의 실행 순서를 조절

## 4. 기아현상
기아현상이란, 특정 프로세스가 필요한 자원을 계속 할당받지 못하고 무한정 기다리는 상태를 말한다. 우선순위가 낮거나, 스케줄러가 자원을 계속 다른 프로세스에 몰아주면서 특정 프로세스는 계속 뒤로 밀려서 실행 기회를 얻지 못하게 되는 현상이다. **SJF, 우선순위 , SRT , MLQ 스케줄링**이 기아현상을 발생시킬 수 있다. 
이러한 기아현상을 해결하기위해 오래 기다린 프로세스의 우선순위를 시간이 지날수록 점점 올려주는 **에이징 기법** 을 사용한 스케줄링 또한 있는데, 그것이 MLFQ, HRN 과 같은 스케줄링이다.
