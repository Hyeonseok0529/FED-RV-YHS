DOM 의 개념을 설명해주세요
HTML 이나 XML 같은 마크업 언어로 작성된 문서를
자바스크립트 같은 프로그래밍 언어가 조작할 수 있도록 하는 인터페이스를 의미합니다.
돔은 계층적 구조를 가진 노드트리로 구성됩니다.
* 노드트리 : 트리구조로 연결된 노드들의 집합
* 트리 : 계층적인 구조를 가지는 자료구조, 하나의 루트 노트에서 시작해 여러개의 자식 노드로 분기됨.
* 노드 : 트리의 개별요소로, 데이터를 저장하며 다른 노드와 연결됨.
=> 노트트리 노드들이 트리 구조로 연결된 형태를 의미

돔이 필요한 이유는 무엇인가요?

동적인 웹 페이지를 구현하기 위해서는 자바스크립트와 같은 프로그래밍 언어가 문서에 접근하고 제어할 수  
있는 수단이 있어야 하지만, 마크업 언어로 작성된 문서에는 이러한 수단이 없습니다.

티비는 있지만 티비를 조작할 수 있는 리모콘은 없는 이치라고 볼 수 있습니다.

따라서 문서에 접근하고 제어할  수 있는 수단인 돔이 필요합니다.

돔을 통해 어떤 동작을 할 수 있는지 예를 들어주세요.
버튼 요소가 클릭되었을 때 , 특정 함수를 호출하도록 이벤트 핸들러를 추가할 수 있고
웹 페이지 내에 새로운 요소를 추가하거나 삭제, 수정할 수 있습니다.

돔은 왜 계층적 구조로 표현하는지 설명해주세요
계층적 구조에서는 노드들 간의 관계가 부모,자식,형제 등으로 명확하게 정의됩니다.
이는 노드의 추가, 제거, 이동 작업을 쉽게 할 수 있도록 도와줍니다.

또한 이벤트가 발생한 요소로부터 이벤트가 올라가는 이벤트 버블링
반대로 이벤트가 내려가는 이벤트 캡처링 동작은
계층적 구조에서 효율적으로 동작하기 때문에 돔을 계층적 구조로 표현하는 것입니다.
